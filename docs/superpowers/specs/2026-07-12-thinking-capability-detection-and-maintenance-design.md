# Thinking 能力探测与 0.8.0 维护性整理设计

## 目标

把“自动关闭思考模式”从服务商名称猜测改为连接测试的真实能力探测，并整理 0.7.0 后新旧实现并存造成的代码与文档歧义。保持个人项目的轻量结构，不引入框架或过度拆分文件。

## 决策

### 自动模式只使用探测结果

- 用户手动选择的 Thinking 策略始终优先，不进行额外探测。
- 自动模式不再根据 API 地址、模型名称或服务商预设选择参数。
- 点击“测试 API”时，后台依次尝试现有的通用 Thinking 控制参数；首个成功策略作为该 API 地址与模型组合的探测结果。
- 探测结果保存在扩展本地设置中，并带有由规范化 API 地址和模型组成的检测键。
- API 地址或模型变化后，检测键不匹配，旧结果自动失效；在重新测试前采用“不发送 Thinking 控制参数”的安全默认值。
- 关闭“自动关闭可控思考模式”时，不探测也不发送相关参数。

探测候选是协议形状列表，不包含服务商判断：

1. `thinking: { type: "disabled" }`
2. `enable_thinking: false`
3. `chat_template_kwargs: { enable_thinking: false }`
4. OpenRouter `reasoning` minimal
5. OpenRouter `reasoning` low
6. 不发送控制参数

候选顺序只影响测试请求数量，不改变用户手动选择能力。

### 错误分类

- 只有响应明确表示 Thinking 参数、字段或值不受支持时，才继续尝试下一个候选。
- 401/403、429、模型不存在、网络错误、超时和服务端错误立即停止，避免一次测试放大为多次无意义请求。
- 服务商若静默忽略未知字段，OpenAI-compatible API 没有通用方式验证其是否真正生效；界面应把结果表述为“连接测试采用的策略”，不承诺模型内部状态。

### 正文翻译与兼容层

- 网页正文继续使用逐段纯文本 SSE，这是唯一主路径。
- `translate_batch` 和 JSON 解析仅作为旧消息兼容层保留一个版本，并在代码中集中标记，不再作为开发文档的默认流程。
- 从设置页 DOM 和表单逻辑移除已经隐藏且对正文无效的 `batchSize`、`maxCharsPerBatch`；共享默认值暂时保留，供旧存储和兼容层读取。
- 不为整理而拆分 `background.js` 或 `content.js`。通过职责分区、纯函数和删除失效 UI 绑定降低理解成本。

### 文档结构

- 新增 `docs/README.md` 作为当前文档索引。
- `docs/development.md` 只描述当前实现、验证命令和维护边界。
- `docs/adr/` 记录仍有效的架构决策。
- `docs/superpowers/specs` 与 `plans` 作为历史设计和实施记录保留，索引中明确其历史属性。
- README、开发文档和 CHANGELOG 不再声称自动模式按服务商名称硬编码，也不再把批量 JSON 描述为正文默认路径。

## 数据流

1. 设置页保存当前表单，并发送 `test_api`。
2. 后台根据手动/自动模式选择单次测试或候选探测。
3. 后台返回连接结果、实际策略和检测键。
4. 设置页把自动探测结果和检测键写入本地存储。
5. 后续正文翻译读取设置；仅当检测键仍匹配时应用已探测策略。

## 测试

至少覆盖：

- 自动模式没有探测结果时不发送 Thinking 参数。
- 自动连接测试按候选推进，并返回首个成功策略。
- 参数不支持错误会继续，鉴权、限流和网络错误不会继续。
- 探测结果仅在 API 地址与模型匹配时生效。
- 手动策略只发送一次测试请求。
- 设置页保存后台返回的探测结果。
- 现有流式、布局、缓存、旧批量兼容和 UI 回归测试继续通过。

## 版本

本次发布为 `0.8.0`。这是行为修正与维护性版本，不创建新的运行时依赖，也不引入数据迁移脚本；旧设置可以继续加载。

# 纯文本流式翻译体验优化设计

> 历史设计记录：其中按服务商名称选择 Thinking 参数的方案已由 0.8.0 的[能力探测设计](2026-07-12-thinking-capability-detection-and-maintenance-design.md)取代。

## 目标

网页翻译不再等待整批 JSON 完成。每个文本段落独立请求纯文本译文，并在模型返回 SSE 文本片段时立即更新页面；默认最多同时翻译两个段落。

本次只优化翻译等待体验，不重构无关的页面识别、译文布局、缓存结构和成本预算。

## 用户体验

- 开始翻译后，当前视口中优先级最高的段落最先进入请求队列。
- 最多同时翻译 `maxConcurrentBatches` 个段落，默认值保持为 2。
- 任意段落收到第一个有效文本片段后，立即用部分译文替换“翻译中...”占位；后续片段持续追加。
- 一个段落完成后立即释放并发槽位并处理下一段，不等待同一轮的其他请求。
- 缓存命中时直接显示完整译文，不建立流式请求。
- 停止翻译或滚动到完全不同的视口时，中止旧视口仍在进行的请求。
- 流式接口不受支持，或在收到任何译文前失败时，自动降级为单段纯文本非流式请求。
- 已经显示部分译文后流中断时，不自动重试以避免重复内容；保留错误状态和点击重试入口。

## 请求协议

网页段落使用 OpenAI-compatible Chat Completions：

- 请求体设置 `stream: true`。
- 输入只包含一个文本段落。
- 提示词要求仅输出译文纯文本，不输出 JSON、Markdown 或解释。
- 后台解析 SSE 的 `data:` 事件，提取 `choices[0].delta.content`。
- `[DONE]` 或响应流正常结束表示完成。

非流式降级使用同一份单段纯文本提示词，读取 `choices[0].message.content`，不使用 JSON。

现有批量 JSON 接口暂时保留，供兼容旧消息、选中文本相关行为和既有回归测试使用；网页正文主链路不再调用它。

## 组件与数据流

### 内容脚本

内容脚本维护一个共享的 `chrome.runtime.Port` 和一个并发受限的段落任务队列。

每个任务包含请求 ID、运行 ID、元素 ID 和原文。后台返回：

- `delta`：追加部分译文并更新占位节点。
- `done`：写入完整译文，将元素标记为完成并释放并发槽位。
- `error`：显示错误和重试入口，并释放并发槽位。
- `cached`：直接显示缓存译文并释放并发槽位。

内容脚本断开 Port 或发送取消消息时，后台中止相关请求。旧运行 ID 的消息一律忽略。

### 后台脚本

后台监听专用 Port，为每个请求创建 `AbortController`：

1. 读取并校验设置。
2. 查询单段缓存。
3. 缓存未命中时发起流式请求。
4. 逐个发送有效文本增量。
5. 完成后保存完整译文到现有缓存，再发送完成事件。
6. Port 断开、任务取消或超时时中止 fetch 和响应流读取。

流式请求使用覆盖整个响应读取过程的超时控制，不能在只收到响应头后就清除计时器。

## 并发与队列

- 不创建 Web Worker；并发指并行的异步 HTTP 请求。
- `maxConcurrentBatches` 保留为存储键以避免配置迁移，语义改为“同时翻译段落数”，范围继续限制为 1～3。
- 调度器采用持续补位方式：任一请求结束后立即启动下一段，不使用 `Promise.all` 批次屏障。
- `batchSize` 和 `maxCharsPerBatch` 不再影响网页正文翻译。设置页隐藏这两个无效字段，但保留历史存储值以兼容旧版本。

## 豆包思考控制

当 API 地址包含 `volces.com`，或模型名包含 `doubao` 时，“自动选择”解析为：

```json
{"thinking":{"type":"disabled"}}
```

用户手动选择的思考策略仍然优先。接口明确报告该参数不受支持时，保留现有的无思考参数重试机制。

## 设置保存与 API 测试

- 设置页继续对表单输入进行防抖自动保存。
- 点击“测试 API”时，继续在测试请求前保存当前完整表单，避免测试成功但配置未落盘。
- 测试请求优先使用单段纯文本流式协议，从而同时验证鉴权、模型、思考参数和 SSE。
- 如果流式能力不受支持但纯文本非流式请求成功，测试仍显示 API 可用，并明确提示实际翻译将使用非流式降级。
- 不新增复杂的永久能力探测表；运行期间只在后台内存中记住当前 URL/模型不支持流式，浏览器服务工作线程重启后允许重新探测。

## 错误处理

- HTTP 401/403：直接报告鉴权错误，不降级。
- 明确的流式参数不支持错误：记住当前 URL/模型并降级为非流式。
- 首个译文增量前的网络失败：允许现有的一次网络重试；仍失败时进行非流式降级。
- 已产生部分译文后的网络失败：停止追加并显示可重试错误，不自动重放请求。
- 超时：中止网络请求并显示现有超时文案。
- 非流式降级也失败：使用现有点击重试交互。

## 测试与验收

实现遵循测试优先，至少覆盖：

1. 火山方舟和 Doubao 自动选择 `thinking.type: disabled`。
2. SSE 数据跨网络分块时仍能正确拼接和解析。
3. `delta` 在 `[DONE]` 前到达内容脚本并更新译文节点。
4. 两个段落并发运行，任一完成后立即启动下一段。
5. 缓存命中不发起 fetch。
6. 切换视口或停止翻译会中止旧请求，并忽略旧增量。
7. 流式不支持时降级到单段纯文本非流式请求。
8. 部分输出后断流不会自动重试或重复文本。
9. 点击“测试 API”会先保存当前设置，并验证流式协议。
10. 现有页面扫描、译文显示、选择翻译和设置页测试保持通过。

验收标准是：模拟慢速 SSE 时，至少一个译文节点必须在响应流结束前进入部分译文状态；全套现有测试和新增回归测试通过。

## 不在本次范围

- 不更换模型或 API 服务商。
- 不修改页面候选文本识别规则。
- 不实现复杂的增量 JSON 解析器。
- 不主动提交或推送 Git。

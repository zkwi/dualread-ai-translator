# 结构感知的网页翻译适配设计

> 历史设计记录：其中 Doubao/火山方舟的固定 Thinking 映射已由 0.8.0 的[能力探测设计](2026-07-12-thinking-capability-detection-and-maintenance-design.md)取代。

## 状态

已确认，目标版本 `0.7.0`。

## 背景

当前页面翻译能快速流式返回，但候选发现和译文插入仍以单个 DOM 元素为中心，缺少对周围布局的统一判断。这造成三类用户可见问题：

- Reddit 信息流的长帖预览把段落设为内联并裁剪，逐段译文被插进裁剪容器后与原文穿插。
- CNN 等多栏首页使用稀疏视口采样，采样点之间的可见标题可能始终没有进入队列。
- 图片标题卡片使用 `flex` 或 `grid` 时，块级译文仍可能成为新列，挤压原文形成近似竖排。

## 目标

- 保持首批最多 4 个高优先级文本的即时 SSE 启动。
- 在首批启动后补齐当前视口内符合条件的标题和正文。
- 根据裁剪、Flex、Grid、列表和 Slot 上下文自动选择安全插入方式。
- 不按 CNN、Reddit 页面 class 不断追加 CSS 补丁。
- 保持页面预算、并发上限、缓存、动态页面扫描和错误重试逻辑。
- 保持火山方舟/Doubao 自动发送 `thinking: { type: "disabled" }`。

## 非目标

- 不引入新的前端框架、依赖或站点配置服务。
- 不尝试重新排版原站全部 DOM。
- 不突破用户配置的请求数、字符数或并发预算。
- 不翻译图片、视频或不可见辅助文本。

## 方案比较

### 方案一：继续增加站点补丁

为 CNN 和 Reddit 分别增加选择器与 CSS。改动小，但站点一改 class 就失效，且无法覆盖其他 Flex/Grid 卡片，因此拒绝。

### 方案二：全局强制译文换行

所有译文都设置更强的块级样式。实现简单，但不能把节点移出裁剪容器，也无法让不换行的 Flex 父容器产生新行，因此拒绝。

### 方案三：结构感知适配层

保留现有候选与流式请求主流程，在候选发现和插入之间增加轻量结构分析，输出统一翻译单元和布局策略。该方案改动集中、可测试、能覆盖同类页面，采用此方案。

## 架构

### 1. 两阶段视口发现

`scanViewport` 继续先用固定采样点找出首批最多 4 个候选并立即排队。首帧后运行一次有界补扫：把采样命中的小型列表或内容区作为局部范围，合并其中所有处于当前视口附近的可读块；如果局部范围不足，再使用现有选择器补扫。所有结果统一去重、按优先级排序，并受 `maxElementsPerScan` 限制。

滚动到新视口时沿用相同流程。补扫不能退化为无界全文 TreeWalker，也不能对长 Reddit 评论区逐元素测量。

### 2. 翻译单元

候选元素在排队前归一化为逻辑翻译单元：

```text
sourceElement  原文状态和唯一 ID 的归属元素
text           实际发送给模型的文本
anchor         译文相对插入的安全锚点
container      译文插入后的直接布局容器
layoutMode     block | flex-row | grid-row | clipped-preview | slotted
```

普通段落的字段都指向自身。检测到裁剪预览时，以预览整体作为 `sourceElement`，文本在配置上限内按完整句子截断，`anchor` 提升到裁剪链接或裁剪容器之外。这样一个预览只产生一个请求和一个译文节点。

### 3. 布局策略

- `block`：在安全锚点后插入整宽译文。
- `flex-row`：保持译文位于合法父节点内，给父节点添加内部数据属性并允许换行；译文使用 `flex: 0 0 100%` 独占下一行。
- `grid-row`：译文设置 `grid-column: 1 / -1`，横跨所有列。
- `clipped-preview`：译文移出 `overflow: hidden/clip`、line-clamp 或受限高度容器。
- `slotted`：继承安全锚点的 `slot`，继续支持 Reddit Web Component 的标题与正文顺序。

内部数据属性只作用于实际包含译文的容器，并在清除翻译时移除，避免永久改变页面布局。

### 4. 流式更新

布局分析只在首次创建译文节点时执行。后续 `loading → streaming → done/error` 复用同一节点，只更新文本和状态，不重复移动 DOM，避免流式片段引发额外布局抖动。

### 5. 请求兼容性

本次不修改 Thinking 策略。`shared.js` 仍将 `volces.com` 或 `doubao` 识别为 `THINKING_DISABLED`，`background.js` 仍生成：

```js
body.thinking = { type: "disabled" };
```

## 失败处理与回退

- 找不到安全锚点时回退到现有元素后插入。
- 布局类型无法识别时使用 `block`，不修改父容器样式。
- 裁剪预览无法得到完整句子边界时，在最大长度前按最后一个空白截断。
- 补扫达到检查或预算上限时停止，后续滚动和动态扫描仍可继续发现内容。

## 非功能要求

- 首批占位不等待补扫完成。
- 不新增网络请求类型或第三方依赖。
- 长 Reddit 线程的 TreeWalker 和布局测量回归上限继续通过。
- 译文不能产生水平溢出，窄卡片的中英文标题宽度不能因译文插入而缩小。
- 清除、停止、远距离滚动取消和内容脚本重载后不能残留布局标记。

## 验收标准

- Reddit 超长信息流预览只在裁剪区外出现一个整宽译文。
- CNN 三栏视口内采样点之间的可见标题也会进入请求队列。
- CNN Underscored 图片标题卡片保持原图片和英文标题宽度，译文位于下一整行。
- 普通段落、列表、Grid、Slot 和动态内容现有测试不回退。
- `npm test` 与 `npm run test:samples` 通过。
- 火山方舟/Doubao 请求体包含 `thinking: { type: "disabled" }`。

# 0.8.0 维护性版本实施计划

## 1. 固化 Thinking 探测契约

- 在共享测试中定义检测键、自动策略失效和手动策略优先行为。
- 在后台测试中定义候选推进、错误中止和测试结果结构。
- 在设置页 UI 测试中定义探测结果持久化行为。

## 2. 实现通用能力探测

- 在 `shared.js` 增加检测键和已探测策略解析纯函数，删除服务商名称猜测。
- 在 `background.js` 增加连接测试候选循环和可继续错误分类。
- 在 `options.js` 保存后台返回的策略与检测键，并在连接字段变化时自然失效。

## 3. 收敛遗留入口

- 删除设置页隐藏批量字段及其表单绑定。
- 保留后台批量 JSON 兼容层，增加明确分区和弃用说明。
- 确认正文路径只发送逐段流式请求。

## 4. 同步当前文档

- 新增文档索引。
- 重写开发说明中的当前翻译流程、Thinking 探测和兼容层边界。
- 更新中英文 README、CHANGELOG 和版本号。

## 5. 验证与发布

- 运行相关单元测试，随后运行 `npm test`。
- 运行真实页面样本；网络拦截站点按现有规则记录，不误判为代码失败。
- 运行公开发布审计，检查 Git 状态与差异。
- 使用简体中文提交信息提交，并推送当前 `main` 分支。

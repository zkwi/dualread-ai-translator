# ADR-0002：通过连接测试探测 Thinking 控制能力

## 状态

Accepted

## 背景

OpenAI-compatible 服务对关闭思考/推理使用不同请求字段。根据服务商域名或模型名称映射参数容易过期，也会把某个服务的临时行为写死到运行时代码。

## 决策

自动模式在用户点击“测试 API”时探测现有通用参数形状，保存首个成功策略及 API/模型检测键。正常翻译只使用仍与检测键匹配的结果；没有匹配结果时不发送扩展字段。用户手动选择始终优先。

只有响应明确表示 Thinking 字段不受支持时才尝试下一个候选，其他错误立即返回。

## 影响

### 正面

- 新服务无需增加域名或模型名称判断。
- 测试连接与实际翻译使用同一请求构造逻辑。
- API/模型变化会自动使旧能力结果失效。

### 负面

- 首次自动测试最坏会产生数个很小的请求。
- 服务商静默忽略未知字段时无法证明参数真正生效。

## 备选方案

- 维护服务商映射表：简单但易过期，拒绝。
- 每次翻译都试错：增加延迟和请求数，拒绝。
- 完全交给用户手动配置：可靠但初次使用成本过高，保留为覆盖自动模式的高级选项。
